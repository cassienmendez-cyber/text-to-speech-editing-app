# Development Roadmap

StoryScribe is delivered in three phases. Each phase builds on the previous one
while preserving the core principle of complete author control.

> **Status:** All three phases are implemented in the current app — capture
> (Phase 1), revision intelligence including batch assistance (Phase 2), and
> editorial intelligence including the character/worldbuilding databases,
> continuity checking, bible↔manuscript linking, and beta-reader roles
> (Phase 3). Remaining future items: EPUB/PDF import and custom note
> categories. See the README's "Implemented" section for specifics.

---

## Phase 1 — MVP

**Goal:** Deliver the fastest and safest way for authors to capture revision
ideas.

Build:

- DOCX import
- TXT import
- Audiobook playback
- Sentence highlighting
- Voice note capture
- Speech transcription
- Audio note preservation
- Comment system
- Flexible note movement
- Offline functionality
- Drive Mode
- Project autosaving

---

## Phase 2 — Revision Intelligence

**Goal:** Help writers organize and implement revisions.

Build:

- Revision dashboard
- Revision passes
- Emotional tags
- Read Like a Reader Mode
- AI suggestion mode
- Batch revision assistance
- Revision history

---

## Phase 3 — Editorial Intelligence

**Goal:** Create an intelligent editorial companion while preserving author
control.

Build:

- Character database
- Worldbuilding database
- Continuity analysis
- Editorial pattern recognition
- Developmental summaries
- Beta reader collaboration
